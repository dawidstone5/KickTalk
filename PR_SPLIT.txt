============================================================
HOW TO SPLIT THE WORKING-TREE CHANGES INTO 4 PRs
============================================================

Current working tree contains four logical changes mixed together:

  PR1  Electron hardening
  PR2  Tokens -> safeStorage
  PR3  CSP + chat sanitize
  PR4  Phase A: Browse / Search / Watch pages + iframe player

This file describes how to land each change as its own PR off `main`.
Delete this file (or add it to .gitignore) once you are done.

------------------------------------------------------------
0. Snapshot the current state so nothing is lost
------------------------------------------------------------

Take a one-shot WIP commit on a throwaway branch. Every PR below will
cherry-pick files / hunks out of this snapshot.

    git checkout -b wip/all-changes
    git add -A
    git commit -m "WIP: all hardening + stream pages"
    # push it as a backup if you want:
    # git push -u origin wip/all-changes

From here on, `wip/all-changes` is your reference tree. Always branch
each PR off `main`.

Tips
------------------------------------------------------------

* `git checkout -p <ref> -- <file>` is the workhorse for splitting
  hunks. Answers: y=keep, n=skip, s=split smaller, e=edit hunk by hand,
  q=quit, ?=help.

* When `e` is needed (a single hunk straddles two PRs), the editor
  opens with the diff. Lines starting with `+` will be added, lines
  starting with `-` will be removed, lines starting with ` ` are
  context. Delete the `+` / `-` lines that DON'T belong in this PR
  (replace with a space prefix to keep them out of the patch).

* If you make a mistake, `git restore --staged --worktree <file>` will
  revert the file to its committed state on the current branch.

* Test each PR in isolation before opening the next. Each PR should
  build and run on its own.

* Each PR's commit message subject should be ~50 chars; body lines
  should wrap at ~72.

* If `gh pr create` is not installed, push the branch and open the PR
  through the GitHub web UI instead.
